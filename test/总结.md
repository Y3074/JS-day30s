### 📁 对应源码
- **路径**：[`1简介.html`](1简介.html) 
> ### 💡 学习直觉（优化版）：
> **入口定位** (引入位置) → **数据解构** (6大类型) → **内存分配** (let/const) → **命名规范** (小驼峰)
### 📌 核心知识地图
1. **引入方式**：
   - **内联/内部**：适合写“临时小样”。
   - **外部引用**：开发主流！将逻辑抽离到 `.js` 文件，保持 HTML 纯净（解耦）。
2. **数据类型**：
   - 基础 6 类：`String`, `Number`, `Boolean`, `undefined`, `null`, `Symbol`。
   - **工具**：使用 `typeof` 随时查看数据的“身份证”。
3. **变量规范**：
   - `let` (会变的) vs `const` (不变的)。
   - 命名：遵循 **camelCase**（小驼峰），拒绝数字开头。

### 🧠 深度勾连与感悟
- **变量的一行式存储**：
  - `let firstName = 'Yuqi', age = 20;` 
  - **勾连点**：这种写法虽然在一行，但它们仍是独立的变量。这为后续学习 **对象 (Object)** 埋下了伏笔——对象能让这些零散信息在逻辑上真正“成团”。
- **内存的理解**：
  - 声明变量 = 申请内存位；赋值 = 填充空间。
  - **勾连 Git**：写代码是填充内存，而 `git commit` 则是将这一刻的“内存状态”永久刻录在时间轴上。
- **区分：内联 (Inline) vs 内部 (Internal) 脚本**
  - **内联 (Inline)**：直接写在 HTML 标签的属性里。
    * *例子*：`<button onclick="alert('点我')">`
    * *感悟*：它是“逻辑与标签”的死捆绑，像**一次性文具**，方便但难以复用。
  - **内部 (Internal)**：写在 `<script>` 标签内，通常位于 `<head>` 或 `<body>` 底部。
    * *例子*：`<script> function sayHi(){...} </script>`
    * *感悟*：它实现了**初步的逻辑分离**。就像在房间里划出了专门的“工作区”，比内联更整洁，适合写这一页专用的逻辑。
  - **深度勾连**：这种从“内联”到“内部”再到“外部”的演变，本质上是程序员在追求**“各司其职”**。HTML 只管骨架，JS 只管灵魂。

### 1️⃣ 数据类型深度细分 (Data Types)
- **分类**：原始数据类型 (Primitive) vs 非原始数据类型 (Non-Primitive)。
- **非原始 (Object, Array)**：
    - **特性**：可变（Mutable），按**引用**比较。
    - **感悟**：即便内容完全一样，`[1,2] === [1,2]` 也是 `false`，因为它们在内存里的“门牌号”不同。只有指向同一个底层对象时才相等。
- **原始 (String, Number, Boolean, Null, Undefined, Symbol)**：
    - **特性**：**不可变性 (Immutability)**。
    - **实验**：`word[0] = 'Y'` 无法修改原字符串。
    - **底层勾连**：重新赋值 `word = 'hello'` 是申请了**新内存**并修改变量指针，而不是在旧地盘上动刀。原始值就像刻在石头上的字，改不了，只能换石头。

---

### 2️⃣ 数学对象 Math (内置全局工具)
- **本质**：Math 是内置对象，提供处理数字的方法。它既不是原始类型，也不是普通的实例。

#### 🎲 随机数边界陷阱 (Off-by-one Error)
- **逻辑**：`Math.floor(Math.random() * 11)`
- **🧠 为什么 * 11？**：
    1. `Math.random()` 范围是 `[0, 1)`，永远拿不到 1。
    2. 如果只 `* 10` 再配合 `Math.floor()`（向下取整），最大只能拿到 9。
    3. **结论**：为了拿到整数 10，必须通过 `* 11` 扩充范围，利用多出的“1”来抵消 `Math.floor` 砍掉的小数位。


#### 常量 vs 方法
- **常量 (属性)**：`Math.PI`, `Math.LN2` (预存的数字，直接拿，快)。
- **方法 (函数)**：`Math.log()`, `Math.sqrt()` (动态加工厂，传参算，慢)。

### 3️⃣ 字符串 (String) 深度方法对比
| 方法 | 功能 | 🧠 深度感悟 (区别) |
| :--- | :--- | :--- |
| **`str[i]` vs `charAt(i)`** | 访问字符 | `charAt` 越界返回 `""` (稳健)；`str[i]` 越界返回 `undefined` (严苛)。 |
| **`+` vs `concat()`** | 拼接 | 无脑选 `+`。具备自动转型魔力，性能更好且语义简洁。 |
| **`substring` vs `substr`** | 截取 | `substring(起点, 终点)` 左闭右开；`substr(起点, 长度)` 已不推荐但在旧代码常见。 |

### 4️⃣ 模板字符串 (Template Strings)

- **价值**：实现了代码的“动静分离”，让动态注入数据不再需要一堆繁杂的引号拼接。

### 📊 常用方法速查手册 (Day 2 简略版)

#### **1. 基础检索与位置**
| 方法 | 功能 | 🧠 核心感悟 |
| :--- | :--- | :--- |
| `.length` | 返回长度 | 它是**属性**，不是方法（不带括号） |
| `[i]` vs `charAt(i)` | 访问字符 | `charAt` 越界返回 `""`，更稳健 |
| `charCodeAt(i)` | 返回 ASCII 码 | 常用于字符加密、大小判断 |
| `indexOf(x)` | 首次出现位置 | 找不到返回 `-1` |
| `lastIndexOf(x)` | 最后出现位置 | 从后往前找，适用于找文件后缀 |
| `search(reg)` | 正则搜索 | 相比 `indexOf`，它能处理复杂的模式匹配 |

#### **2. 内容检查 (返回布尔值)**
| 方法 | 功能 | 🧠 场景勾连 |
| :--- | :--- | :--- |
| `includes(x)` | 是否包含 | 最通用的判断方法 |
| `startsWith(x)` | 是否开头 | 判断 URL 是否以 `https` 开头 |
| `endsWith(x)` | 是否结尾 | 判断文件名是否以 `.jpg` 结尾 |

#### **3. 修改与变换 (生成新串)**
| 方法 | 功能 | 🧠 场景勾连 |
| :--- | :--- | :--- |
| `toUpperCase()` | 转大写 | 统一格式（如验证码对比） |
| `toLowerCase()` | 转小写 | 搜索关键词不区分大小写处理 |
| `trim()` | 去两端空格 | 必做！清理用户输入的脏数据 |
| `replace(a, b)` | 替换首个 | `replace('1','2')` 只换第一个 |
| `replaceAll(a, b)` | 全量替换 | 替换掉字符串里所有的匹配项 |
| `repeat(n)` | 重复 n 次 | 快速生成分割线或占位符 |

#### **4. 截取与拆分**
| 方法 | 功能 | 🧠 核心差异 |
| :--- | :--- | :--- |
| `split(x)` | **转为数组** | **Day 2 最强方法**，将字符串结构化 |
| `substring(a, b)` | 截取区间 | `[a, b)` 左闭右开，不支持负数 |
| `slice(a, b)` | 截取/切片 | **最推荐**，支持负数（如 `-1` 取最后一位） |
| `match(reg)` | 正则提取 | 返回一个包含所有匹配结果的数组 |



#### **5. 数字 (Number) 与转换方法**
| 转换方式 | 示例 | 🧠 备注 (场景) |
| :--- | :--- | :--- |
| `parseInt(str)` | `parseInt('10.5')` $\rightarrow$ `10` | 丢弃小数位，只取整数部分 |
| `parseFloat(str)` | `parseFloat('10.5')` $\rightarrow$ `10.5` | 保留小数，常用于经纬度或价格 |
| `Number(str)` | `Number('123')` $\rightarrow$ `123` | 标准强制类型转换 |
| **`+` (加号)** | `+'100'` $\rightarrow$ `100` | **最推荐！** 隐式转换，代码最简洁 |
| `isNaN(x)` | `isNaN('abc')` $\rightarrow$ `true` | 检查一个值是否“不是数字” |

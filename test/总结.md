  - `let firstName = 'Yuqi', age = 20;` 
  - **勾连点**：这种写法虽然在一行，但它们仍是独立的变量。这为后续学习 **对象 (Object)** 埋下了伏笔——对象能让这些零散信息在逻辑上真正“成团”。
- **内存的理解**：
  - 声明变量 = 申请内存位；赋值 = 填充空间。
  - **勾连 Git**：写代码是填充内存，而 `git commit` 则是将这一刻的“内存状态”永久刻录在时间轴上。
- **区分：内联 (Inline) vs 内部 (Internal) 脚本**
  - **内联 (Inline)**：直接写在 HTML 标签的属性里。
    * *例子*：`<button onclick="alert('点我')">`
    * *感悟*：它是“逻辑与标签”的死捆绑，像**一次性文具**，方便但难以复用。
  - **内部 (Internal)**：写在 `<script>` 标签内，通常位于 `<head>` 或 `<body>` 底部。
    * *例子*：`<script> function sayHi(){...} </script>`
    * *感悟*：它实现了**初步的逻辑分离**。就像在房间里划出了专门的“工作区”，比内联更整洁，适合写这一页专用的逻辑。
  - **深度勾连**：这种从“内联”到“内部”再到“外部”的演变，本质上是程序员在追求**“各司其职”**。HTML 只管骨架，JS 只管灵魂。

### 📌 核心知识地图
1. **数据类型的两极分化**：
   - **原始 (Primitive)**：`String`, `Number`, `Boolean`, `Null`, `Undefined`, `Symbol`。
   - **非原始 (Non-Primitive)**：`Object`, `Array`。
2. **数学对象 Math (内置全局工具)**：
   - **常量**：`Math.PI`, `Math.LN2` (系统预存的属性，直接拿)。
   - **方法**：`Math.log()`, `Math.sqrt()`, `Math.floor()`, `Math.ceil()`, `Math.round()` (动态加工厂，需传参)。
3. **模板字符串 (Template Strings)**：
   - **语法**：使用反引号 **`` ` ``** 和 `${expression}`。
   - **价值**：实现“动静分离”，告别繁琐的引号与 `+` 拼接，支持多行文本。

   - **VS Code**：必须通过 `File -> Open Folder` 打开项目，否则 Live Server 无法定位根目录。

### 🧠 深度勾连与感悟
- **内存的死规矩：原始类型不可变 (Immutability)**：
  - **实验**：试图 `word[0] = 'Y'` 是无效的。
  - **感悟**：原始数据一旦生成，内部结构就锁死了。重新赋值 `word = 'hello'` 本质是申请了**新内存**并修改了变量指针，而不是在旧地盘上动刀。
- **非原始类型的“门牌号”理论**：
  - **现象**：即便内容完全一样，`[1,2] === [1,2]` 也是 `false`。
  - **感悟**：非原始类型是按**引用（内存地址）**比较的，只有两个变量指向同一个底层对象时才相等。
- **随机数边界陷阱 (Off-by-one Error)**：
  - **公式**：`Math.floor(Math.random() * 11)` (生成 0-10 的整数)
  - **为什么 * 11？**：因为 `Math.random()` 永远拿不到 1。如果不乘以 11 而是乘以 10，再配合向下取整，最大只能拿到 9。多出的“1”是为了抵消 `floor` 砍掉的小数位。
- **字符串访问与拼接的优雅选择**：
  - **访问**：`charAt(i)` 越界返回 `""`（稳健）；`str[i]` 越界返回 `undefined`（严苛）。
  - **拼接**：无脑选 `+`。具备自动转型魔力，性能优于 `concat()`。

<summary><b>📅 Day 03：布尔逻辑、操作符与日期对象 (Date)</b></summary>

### 📁 对应源码
- **路径**：[`3布尔运算符日期.html`](3布尔运算符日期.html) 

> ### 💡 学习直觉：
> **真假判定** (布尔真假值) → **状态区分** (未定义与无效) → **逻辑加工** (操作符) → **交互体验** (窗口方法) → **时间坐标** (Date 对象)

### 📌 核心知识地图
1. **布尔值与判定**：
   - **真值 (Truthy)**：所有的数无论正数还是负数（唯独0除外）、除了空字符串以外的所有字符、`true`。
   - **假值 (Falsy)**：`0`, `null`, `undefined`, `NaN`, `false`, `""` (空串)。
2. **变量状态**：
   - **未定义 (Undefined)**：只是声明变量且不赋值。
   - **无效 (Null)**：空值。
3. **操作符全解**：
   - **算术/赋值**：`+`, `-`, `*`, `/`, `%`, `**` 及 `=`, `+=`, `-=`, `%=`, `**=`。
   - **比较**：`==`, `!=`, `>`, `<`, `>=`, `<=`。
   - **逻辑**：`&&` (与), `||` (或), `!` (非)。
   - **增量**：`++`, `--`。
   - **三元**：`条件 ? 表达式1 : 表达式2`。
4. **窗口方法**：
   - `alert(message)`：窗口提示方法。
   - `prompt('required text', 'optional text')`：窗口提示词方法。
   - `confirm(msg)`：窗口确认方法，返回值 true 或 false。
5. **日期对象 Date**：
   - `new Date()`：创建时间对象。
   - 获取值：`getFullYear()`, `getMonth()`, `getDate()`, `getDay()`, `getHours()`, `getMinutes()`, `getSeconds()`。
   - 获取时间戳：`Date.now()` 或 `new Date().getTime()`。

### 🧠 深度勾连与感悟
- **真假值的隐式转换**：
  - **勾连点**：在条件判断中，记住 6 个假值（0, null, undefined, NaN, false, ""），剩下的全都是真值，这能极大简化逻辑判断。
- **未定义 (Undefined) vs 无效 (Null)**：
  - `undefined` 是系统默认状态（只声明未赋值），而 `null` 是明确的“空值”。
- **三元算符的优雅**：
  - `isRaining ? console.log('You need a rain coat.') : console.log('No need for a rain coat.')`
  - **感悟**：本质上是 `if-else` 的单行替代品，让代码更加紧凑。
- **时间戳的“绝对坐标”**：
  - `Date.now()` 获取的是从 1970年1月1日 至今的毫秒数。
  - **深度勾连**：时间戳在系统中是绝对唯一的，常用来做时间差计算（例如：`console.log(allSeconds == timeInSeconds)` 验证时间的同步）或生成唯一标识。

<summary><b>📅 Day 04：条件句 (Conditionals) 与程序的“十字路口”</b></summary>

### 📁 对应源码
- **路径**：[`4条件句.html`](4条件句.html)

> ### 💡 学习直觉：
> **逻辑评估** (布尔真假) → **单线/双线分支** (if / if-else) → **多线路由** (else if / switch) → **极简表达** (三元运算)

### 📌 核心知识地图
1. **控制流机制**：
   - **条件执行**：如果某个表达式为真（Truthy），则执行特定的语句块。这是程序做出“决策”的基础。
   - **重复执行**：只要某个表达式为真，一个或多个语句就会被重复执行。
2. **四大条件分流方式**：
   - `if`：单一条件把关。
   - `if else`：非此即彼的二选一。
   - `if else if else`：多区间、多维度的层层筛选。
   - `switch`：针对单一变量的具体值精确匹配。
3. **三元算符**：
   - 语法：`条件 ? 真执行 : 假执行`

### 🧠 深度勾连与感悟
- **三元运算符的绝对主力地位**：
  ```javascript
  let isRaining = true
  isRaining ? console.log('yes') : console.log('no')
  ```
  - **感悟**：在日后开发大型 Web 项目时，三元运算符是按条件渲染页面的绝对主力，必须练就一眼看穿它的条件反射。
- **`if-else` vs `switch` 的实战抉择**：
  - **打范围战**：当判断条件是区间（如 `age >= 18`）时，必须用 `if else if`。
  - **打点位战**：当判断条件是固定的几个具体值（如判断红绿灯的红/黄/绿、星期一到星期日）时，用 `switch` 结构更清晰，且在底层执行效率通常更高。
- **Switch 的“瀑布穿透”陷阱 (Fall-through)**：
  - **底层避坑**：在 `switch` 中匹配中某个 `case` 后，如果没有写 `break` 及时刹车，程序会无视后面的条件，像瀑布一样向下砸，把后续所有 `case` 里的代码全都执行一遍。这是初学者必踩的坑。

<summary><b>📅 Day 05：数组 (Arrays) —— 内存空间的“储物柜”与高效操控</b></summary>

### 📁 对应源码
- **路径**：[`5数组.html`](5数组.html)

> ### 💡 学习直觉：
> **容器创建** (创建空/实值数组) → **结构拆分** (split 转化) → **全能兵器库** (16种操控方法) → **内存引用** (索引与地址)

### 📌 核心知识地图
1. **数组本质**：
   - 数组可存储多个、不同类型的值。
   - 每个值对应一个索引（Index），且在内存地址中存在引用（Reference）。
2. **创建方式**：
   - **构造器法**：`new Array(size)`（预设大小）。
   - **方括号法**：`[]`（最推荐，内部可混搭不同数据类型）。
   - **分裂法**：`str.split('')` 将字符串瞬间“结构化”为数组。
3. **高频操控兵器库 (16法)**：
   - **定位/检查**：`indexOf`, `lastIndexOf`, `includes`, `isArray`。
   - **增删改**：`push`/`pop` (尾部), `shift`/`unshift` (头部), `fill` (填充)。
   - **结构变换**：`toString`, `join` (数组转字符), `concat` (连接), `reverse` (反转), `sort` (排序)。
   - **切片与拼接**：`slice` (无损截取), `splice` (威力巨大的“手术刀”，支持增删改)。

### 🧠 深度勾连与感悟
- **内存地址的“储物柜”哲学**：
  - **感悟**：数组变量存的是“钥匙（地址）”，索引是“格子编号”。修改数组内容只是在同一个地址里“更换家具”，所以即使是 `const` 定义的数组也能被修改。
- **性能权衡：为什么数组可变？**：
  - **勾连点**：如果是百万级数据，修改一个元素就要全量复制（不可变），电脑会直接卡死。JS 允许数组原地（In-place）修改，是空间与速度的最佳平衡。
- **“无损” slice vs “有损” splice**：
  - **避坑点**：`slice` 只是复印了一份，原数组不动；而 `splice` 是直接在原数组上“动刀”，会改变原数组。使用前必须想清楚是否要保留原数据。
- **split 与 join 的“镜像映射”**：
  - **感悟**：`split` 让字符串变数组（解构），`join` 让数组变回字符串（重组）。它们是一对反义词，常配合使用来处理复杂的文本替换任务。
- **头尾操作的性能差异**：
  - **底层认知**：`push/pop` 只需要动尾巴，很快；而 `shift/unshift` 因为要让后面所有元素搬家（重新排索引），在数据量极大时性能消耗更高。

<summary><b>📅 Day 06：循环 (Loops) —— 自动化流水线的“调度艺术”</b></summary>

### 📁 对应源码
- **路径**：[`6循环.html`](6循环.html)

> ### 💡 学习直觉：
> **标准控制** (for 循环) → **条件驱动** (while 家族) → **现代遍历** (for...of) → **精准干扰** (break/continue)

### 📌 核心知识地图
1. **标准 for 循环**：
   - 结构：`for(初始化; 条件; 步进)`。适用于**已知次数**的精确操控。
2. **While 家族 (条件驱动)**：
   - **`while`**：先判断，再执行。如果初始条件不成立，一次都不会跑。
   - **`do while`**：先执行，再判断。**至少会执行一次**，无论条件真假。
3. **现代遍历 (`for...of`)**：
   - 语法：`for (const element of arr) { ... }`。处理数组最优雅、可读性最高的方式。
4. **循环控制**：
   - `break`：立即终止并跳出整个循环。
   - `continue`：跳过当前这一轮，直接开始下一轮。

---

### 🔍 深度对比：我该选哪个循环？

#### 1️⃣ While vs Do While：谁更“鲁莽”？
* **While (温和的门卫)**：先看门票，再让进屋。逻辑是 `判断 -> 执行`。
* **Do While (鲁莽的保镖)**：先进屋，再补票。逻辑是 `执行 -> 判断`。
* **场景建议**：如果你需要程序**至少运行一次**（如：先显示菜单，再根据用户输入决定是否退出），必选 `do while`。



#### 2️⃣ For vs While：精确制导 vs 巡航搜索
| 特性 | For 循环 (精确制导) | While 循环 (巡航搜索) |
| :--- | :--- | :--- |
| **驱动核心** | **计数器驱动**。三要素写在一行，结构紧凑。 | **条件驱动**。只要条件为真就一直跑。 |
| **适用场景** | **已知次数**。遍历数组、跑固定次数。 | **未知次数**。等待某个事件发生或用户输入。 |
| **变量寿命** | 计数器 `i` 随循环结束而销毁。 | 变量通常声明在外部，可能污染全局。 |



---

### 🧠 深度勾连与感悟
- **自动化流水线的本质**：
  - **感悟**：数组是“货架”，循环就是“扫码枪”。它把重复的体力活（如处理 10,000 条数据）交给 CPU，实现了从“人工”到“自动化”的跨越。
- **内存与作用域的暗示**：
  - **勾连点**：`for (const element of arr)` 里的 `const` 说明每一轮迭代都是一个独立的块级作用域，这保证了数据的安全性和内存的及时回收。
- **`break` 与 `continue` 的战术运用**：
  - **感悟**：`break` 是“止损”，发现目标就撤；`continue` 是“过滤”，遇到垃圾数据就跳过。灵活组合能极大地优化 CPU 执行效率。
- **避坑：死循环的恐怖**：
  - **提醒**：在 `while` 中一定记得更新条件（如 `i++`），否则 CPU 会瞬间拉满。

> ### 💡 学习直觉：
> **逻辑封装** (声明/表达式) → **参数操控** (arguments/Rest) → **匿名/自调用** (隔离与回调) → **现代方案** (箭头函数/默认参数)

### 📌 核心知识地图
1. **函数本质**：设计用来执行特定任务的可重用代码块。要求：简洁易读、可重复、易于测试。
2. **五种声明/创建形态**：
   - **函数声明**：`function f(){}`，支持变量提升（先调用后定义）。
   - **表达式函数**：将函数赋给变量 `const f = function(){}`，逻辑更严谨。
   - **匿名函数**：没有名字的“临时工”，常作为回调或表达式的值。
   - **自调用函数 (IIFE)**：`(function(){})()`，定义即执行，创建私有作用域，防止变量污染。
   - **箭头函数**：`() => {}`，现代 JS 标准，语法简洁，不绑定 `arguments`。
3. **参数处理的“双雄会”**：
   - **`arguments` 对象**：正则函数自带。**类数组对象**（非真数组），无法直接用数组方法。
   - **`...args` (剩余参数)**：箭头函数标配。**真正的数组**，可以使用 `sort`, `map` 等所有数组兵器。
4. **防御性编程**：
   - **默认参数**：`name = 'Peter'`。当调用者没给参数时，函数依然能稳健运行。

---

### 🔍 深度对比：拨开函数的迷雾

#### ⚡ `arguments` vs `...args`：谁才是真数组？

* **真相**：`typeof arguments` 是 `object`，但 `Array.isArray(arguments)` 是 `false`。
* **输入陷阱**：`sumAll([1,2])` 传入的是一个包装好的数组参数；`sumAll(1,2)` 传入的是两个独立的数字参数。

#### 🛡️ 表达式函数 vs 自调用函数 (IIFE)
* **函数表达式**：是**“抽油烟机”**，安好了不按开关（调用）不转。
* **自调用函数**：是**“烟花”**，点火（定义）即爆炸（执行），执行完即销毁。
* **本质区别**：表达式函数为了**复用**；自调用函数为了**隔离**（保护内部变量不被外界窥探）。

---

### 🧠 深度勾连与感悟
- **“内容与容器”的哲学**：
  - **感悟**：匿名函数是“内容”，表达式函数是“容器”。我们给“无名氏”安一个变量名，是为了在内存地址中给它留个位子，方便反复调用。
- **函数的“血统”鉴定**：
  - **勾连点**：使用 `Array.isArray()` 是识破类数组对象（如 `arguments`）伪装的唯一准则。
- **为什么箭头函数没有 `arguments`？**：
  - **感悟**：这是 JS 引擎的优化。箭头函数设计初衷就是轻量化。如果需要处理无限参数，强制要求使用更强大的 `...args` 真数组，这其实是倒逼开发者写出更规范的代码。
- **默认参数的稳定性**：
  - **感悟**：默认参数不仅是图省事，它更像是一种“合同保障”——确保函数内部在处理 `undefined` 时不会崩溃。

<summary><b>📅 Day 08：作用域与对象 (Scope & Objects) —— 划定地盘与数据打包</b></summary>

### 📁 对应源码
- **路径**：[`8对象与作用域.html`](8对象.html)

> ### 💡 学习直觉：
> **作用域墙** (var vs let) → **数据实体** (键值对构建) → **精准访问** (点语法 vs 方括号) → **格式转换** (Object 核心方法)

### 📌 核心知识地图

#### 1. 作用域 (Scope)：变量的“活动地盘”
* **全局作用域 (Global)**：声明在所有代码块外部，哪里都能访问。
* **局部作用域 (Local)**：被包裹在特定区域内。
* **⚠️ 关键字的“墙”**：
    * **`var` (函数作用域)**：只有 `function(){}` 能关住它，无视普通的 `{}`（如 if/for），容易造成变量污染。
    * **`let/const` (块级作用域)**：现代 JS 规范，严格被限制在 `{}` 内部，安全可靠。

#### 2. 对象 (Objects)：数据的“终极打包器”
* **创建**：`const obj = {}`。对象由“键 (Key)”和“值 (Value)”组成。
* **访问值的两套法则**：
    * **点语法 (`.`)**：`person.firstName`。适用于**规矩**的、无特殊字符的已知键名。
    * **方括号语法 (`['']`)**：`person['phone number']`。专治**“叛逆”**键名（如带空格、数字开头）或**动态变量**。

#### 3. 对象的操控与进阶方法
* **增改属性**：直接赋值即可，如 `person.isMarried = true` 或 `person.skills.push('Java')`。
* **四大法宝 (Object Methods)**：
    * `Object.assign({}, obj)`：**无损克隆**。把原对象的属性拷贝到一个全新的 `{}` 纸箱里，保护原数据。
    * `Object.keys(obj)`：抽取所有**键**，返回一个一维数组。
    * `Object.entries(obj)`：抽取**键和值**，返回一个二维数组 `[ [key, value], ... ]`。
    * `obj.hasOwnProperty('key')`：安全检查。判断对象肚子里到底有没有这个属性（返回 true/false）。

---

### 🧠 深度勾连与感悟

* **告别 `var` 的时代**：
    * **感悟**：理解了 `var` 的“穿墙术”，就能明白为什么现代前端开发要全量拥抱 `let` 和 `const`。把变量锁在它该呆的代码块里，是减少 Bug 的第一要义。
* **为什么 `'phone number'` 要加引号？**：
    * **避坑点**：JS 对象的键底层都是字符串。只要键名里出现了空格、连字符等“不合法”的变量命名符号，就必须穿上引号防弹衣，并且后续只能用 `['']` 来召唤它。
* **`Object.entries()` 的本质是“翻译”而非“复制”**：
    * **勾连点 (联动 Day 06)**：对象是铁板一块，没法直接用循环遍历。`entries()` 的伟大之处在于它把对象“拆解并翻译”成了二维数组，这样我们就能无缝接入 `for...of` 来批量处理对象里的数据了！

  - `let firstName = 'Yuqi', age = 20;` 
  - **勾连点**：这种写法虽然在一行，但它们仍是独立的变量。这为后续学习 **对象 (Object)** 埋下了伏笔——对象能让这些零散信息在逻辑上真正“成团”。
- **内存的理解**：
  - 声明变量 = 申请内存位；赋值 = 填充空间。

- **区分：内联 (Inline) vs 内部 (Internal) 脚本**

    * *感悟*：它是“逻辑与标签”的死捆绑，像**一次性文具**，方便但难以复用。
  - **内部 (Internal)**：写在 `<script>` 标签内，通常位于 `<head>` 或 `<body>` 底部。
    * *例子*：`<script> function sayHi(){...} </script>`
    * *感悟*：它实现了**初步的逻辑分离**。就像在房间里划出了专门的“工作区”，比内联更整洁，适合写这一页专用的逻辑。
  - **深度勾连**：这种从“内联”到“内部”再到“外部”的演变，本质上是程序员在追求**“各司其职”**。HTML 只管骨架，JS 只管灵魂。

### 📌 核心知识地图
1. **数据类型的两极分化**：
   - **原始 (Primitive)**：`String`, `Number`, `Boolean`, `Null`, `Undefined`, `Symbol`。
   - **非原始 (Non-Primitive)**：`Object`, `Array`。
2. **数学对象 Math (内置全局工具)**：
   - **常量**：`Math.PI`, `Math.LN2` (系统预存的属性，直接拿)。
   - **方法**：`Math.log()`, `Math.sqrt()`, `Math.floor()`, `Math.ceil()`, `Math.round()` (动态加工厂，需传参)。
3. **模板字符串 (Template Strings)**：
   - **语法**：使用反引号 **`` ` ``** 和 `${expression}`。
   - **价值**：实现“动静分离”，告别繁琐的引号与 `+` 拼接，支持多行文本。

   - **VS Code**：必须通过 `File -> Open Folder` 打开项目，否则 Live Server 无法定位根目录。

### 🧠 深度勾连与感悟
- **内存的死规矩：原始类型不可变 (Immutability)**：
  - **实验**：试图 `word[0] = 'Y'` 是无效的。
  - **感悟**：原始数据一旦生成，内部结构就锁死了。重新赋值 `word = 'hello'` 本质是申请了**新内存**并修改了变量指针，而不是在旧地盘上动刀。
- **非原始类型的“门牌号”理论**：
  - **现象**：即便内容完全一样，`[1,2] === [1,2]` 也是 `false`。
  - **感悟**：非原始类型是按**引用（内存地址）**比较的，只有两个变量指向同一个底层对象时才相等。
- **随机数边界陷阱 (Off-by-one Error)**：
  - **公式**：`Math.floor(Math.random() * 11)` (生成 0-10 的整数)
  - **为什么 * 11？**：因为 `Math.random()` 永远拿不到 1。如果不乘以 11 而是乘以 10，再配合向下取整，最大只能拿到 9。多出的“1”是为了抵消 `floor` 砍掉的小数位。
- **字符串访问与拼接的优雅选择**：
  - **访问**：`charAt(i)` 越界返回 `""`（稳健）；`str[i]` 越界返回 `undefined`（严苛）。
  - **拼接**：无脑选 `+`。具备自动转型魔力，性能优于 `concat()`。

<summary><b>📅 Day 03：布尔逻辑、操作符与日期对象 (Date)</b></summary>

### 📁 对应源码

> ### 💡 学习直觉：

1. **布尔值与判定**：
   - **真值 (Truthy)**：所有的数无论正数还是负数（唯独0除外）、除了空字符串以外的所有字符、`true`。
   - **假值 (Falsy)**：`0`, `null`, `undefined`, `NaN`, `false`, `""` (空串)。
2. **变量状态**：
   - **未定义 (Undefined)**：只是声明变量且不赋值。
   - **无效 (Null)**：空值。
3. **操作符全解**：
   - **算术/赋值**：`+`, `-`, `*`, `/`, `%`, `**` 及 `=`, `+=`, `-=`, `%=`, `**=`。
   - **比较**：`==`, `!=`, `>`, `<`, `>=`, `<=`。
   - **逻辑**：`&&` (与), `||` (或), `!` (非)。
   - **增量**：`++`, `--`。
   - **三元**：`条件 ? 表达式1 : 表达式2`。
4. **窗口方法**：
   - `alert(message)`：窗口提示方法。
   - `prompt('required text', 'optional text')`：窗口提示词方法。
   - `confirm(msg)`：窗口确认方法，返回值 true 或 false。
5. **日期对象 Date**：
   - `new Date()`：创建时间对象。
   - 获取值：`getFullYear()`, `getMonth()`, `getDate()`, `getDay()`, `getHours()`, `getMinutes()`, `getSeconds()`。
   - 获取时间戳：`Date.now()` 或 `new Date().getTime()`。

### 🧠 深度勾连与感悟
- **真假值的隐式转换**：
  - **勾连点**：在条件判断中，记住 6 个假值（0, null, undefined, NaN, false, ""），剩下的全都是真值，这能极大简化逻辑判断。
- **未定义 (Undefined) vs 无效 (Null)**：
  - `undefined` 是系统默认状态（只声明未赋值），而 `null` 是明确的“空值”。
- **三元算符的优雅**：
  - `isRaining ? console.log('You need a rain coat.') : console.log('No need for a rain coat.')`
  - **感悟**：本质上是 `if-else` 的单行替代品，让代码更加紧凑。
- **时间戳的“绝对坐标”**：
  - `Date.now()` 获取的是从 1970年1月1日 至今的毫秒数。
  - **深度勾连**：时间戳在系统中是绝对唯一的，常用来做时间差计算（例如：`console.log(allSeconds == timeInSeconds)` 验证时间的同步）或生成唯一标识。

<summary><b>📅 Day 04：条件句 (Conditionals) 与程序的“十字路口”</b></summary>

### 📁 对应源码
- **路径**：[`4条件句.html`](4条件句.html)

> ### 💡 学习直觉：
> **逻辑评估** (布尔真假) → **单线/双线分支** (if / if-else) → **多线路由** (else if / switch) → **极简表达** (三元运算)

### 📌 核心知识地图
1. **控制流机制**：
   - **条件执行**：如果某个表达式为真（Truthy），则执行特定的语句块。这是程序做出“决策”的基础。
   - **重复执行**：只要某个表达式为真，一个或多个语句就会被重复执行。
2. **四大条件分流方式**：
   - `if`：单一条件把关。
   - `if else`：非此即彼的二选一。
   - `if else if else`：多区间、多维度的层层筛选。
   - `switch`：针对单一变量的具体值精确匹配。
3. **三元算符**：
   - 语法：`条件 ? 真执行 : 假执行`

### 🧠 深度勾连与感悟
- **三元运算符的绝对主力地位**：
  ```javascript
  let isRaining = true
  isRaining ? console.log('yes') : console.log('no')
  ```
  - **感悟**：在日后开发大型 Web 项目时，三元运算符是按条件渲染页面的绝对主力，必须练就一眼看穿它的条件反射。
- **`if-else` vs `switch` 的实战抉择**：
  - **打范围战**：当判断条件是区间（如 `age >= 18`）时，必须用 `if else if`。
  - **打点位战**：当判断条件是固定的几个具体值（如判断红绿灯的红/黄/绿、星期一到星期日）时，用 `switch` 结构更清晰，且在底层执行效率通常更高。
- **Switch 的“瀑布穿透”陷阱 (Fall-through)**：
  - **底层避坑**：在 `switch` 中匹配中某个 `case` 后，如果没有写 `break` 及时刹车，程序会无视后面的条件，像瀑布一样向下砸，把后续所有 `case` 里的代码全都执行一遍。这是初学者必踩的坑。

<summary><b>📅 Day 05：数组 (Arrays) —— 内存空间的“储物柜”与高效操控</b></summary>

### 📁 对应源码
- **路径**：[`5数组.html`](5数组.html)

> ### 💡 学习直觉：
> **容器创建** (创建空/实值数组) → **结构拆分** (split 转化) → **全能兵器库** (16种操控方法) → **内存引用** (索引与地址)

### 📌 核心知识地图
1. **数组本质**：
   - 数组可存储多个、不同类型的值。
   - 每个值对应一个索引（Index），且在内存地址中存在引用（Reference）。
2. **创建方式**：
   - **构造器法**：`new Array(size)`（预设大小）。
   - **方括号法**：`[]`（最推荐，内部可混搭不同数据类型）。
   - **分裂法**：`str.split('')` 将字符串瞬间“结构化”为数组。
3. **高频操控兵器库 (16法)**：
   - **定位/检查**：`indexOf`, `lastIndexOf`, `includes`, `isArray`。
   - **增删改**：`push`/`pop` (尾部), `shift`/`unshift` (头部), `fill` (填充)。
   - **结构变换**：`toString`, `join` (数组转字符), `concat` (连接), `reverse` (反转), `sort` (排序)。
   - **切片与拼接**：`slice` (无损截取), `splice` (威力巨大的“手术刀”，支持增删改)。

### 🧠 深度勾连与感悟
- **内存地址的“储物柜”哲学**：
  - **感悟**：数组变量存的是“钥匙（地址）”，索引是“格子编号”。修改数组内容只是在同一个地址里“更换家具”，所以即使是 `const` 定义的数组也能被修改。
- **性能权衡：为什么数组可变？**：
  - **勾连点**：如果是百万级数据，修改一个元素就要全量复制（不可变），电脑会直接卡死。JS 允许数组原地（In-place）修改，是空间与速度的最佳平衡。
- **“无损” slice vs “有损” splice**：
  - **避坑点**：`slice` 只是复印了一份，原数组不动；而 `splice` 是直接在原数组上“动刀”，会改变原数组。使用前必须想清楚是否要保留原数据。
- **split 与 join 的“镜像映射”**：
  - **感悟**：`split` 让字符串变数组（解构），`join` 让数组变回字符串（重组）。它们是一对反义词，常配合使用来处理复杂的文本替换任务。
- **头尾操作的性能差异**：
  - **底层认知**：`push/pop` 只需要动尾巴，很快；而 `shift/unshift` 因为要让后面所有元素搬家（重新排索引），在数据量极大时性能消耗更高。

<summary><b>📅 Day 06：循环 (Loops) —— 自动化流水线的“调度艺术”</b></summary>

### 📁 对应源码
- **路径**：[`6循环.html`](6循环.html)

> ### 💡 学习直觉：
> **标准控制** (for 循环) → **条件驱动** (while 家族) → **现代遍历** (for...of) → **精准干扰** (break/continue)

### 📌 核心知识地图
1. **标准 for 循环**：
   - 结构：`for(初始化; 条件; 步进)`。适用于**已知次数**的精确操控。
2. **While 家族 (条件驱动)**：
   - **`while`**：先判断，再执行。如果初始条件不成立，一次都不会跑。
   - **`do while`**：先执行，再判断。**至少会执行一次**，无论条件真假。
3. **现代遍历 (`for...of`)**：
   - 语法：`for (const element of arr) { ... }`。处理数组最优雅、可读性最高的方式。
4. **循环控制**：
   - `break`：立即终止并跳出整个循环。
   - `continue`：跳过当前这一轮，直接开始下一轮。

---

### 🔍 深度对比：我该选哪个循环？

#### 1️⃣ While vs Do While：谁更“鲁莽”？
* **While (温和的门卫)**：先看门票，再让进屋。逻辑是 `判断 -> 执行`。
* **Do While (鲁莽的保镖)**：先进屋，再补票。逻辑是 `执行 -> 判断`。
* **场景建议**：如果你需要程序**至少运行一次**（如：先显示菜单，再根据用户输入决定是否退出），必选 `do while`。



#### 2️⃣ For vs While：精确制导 vs 巡航搜索
| 特性 | For 循环 (精确制导) | While 循环 (巡航搜索) |
| :--- | :--- | :--- |
| **驱动核心** | **计数器驱动**。三要素写在一行，结构紧凑。 | **条件驱动**。只要条件为真就一直跑。 |
| **适用场景** | **已知次数**。遍历数组、跑固定次数。 | **未知次数**。等待某个事件发生或用户输入。 |
| **变量寿命** | 计数器 `i` 随循环结束而销毁。 | 变量通常声明在外部，可能污染全局。 |



---

### 🧠 深度勾连与感悟
- **自动化流水线的本质**：
  - **感悟**：数组是“货架”，循环就是“扫码枪”。它把重复的体力活（如处理 10,000 条数据）交给 CPU，实现了从“人工”到“自动化”的跨越。
- **内存与作用域的暗示**：
  - **勾连点**：`for (const element of arr)` 里的 `const` 说明每一轮迭代都是一个独立的块级作用域，这保证了数据的安全性和内存的及时回收。
- **`break` 与 `continue` 的战术运用**：
  - **感悟**：`break` 是“止损”，发现目标就撤；`continue` 是“过滤”，遇到垃圾数据就跳过。灵活组合能极大地优化 CPU 执行效率。
- **避坑：死循环的恐怖**：
  - **提醒**：在 `while` 中一定记得更新条件（如 `i++`），否则 CPU 会瞬间拉满。

> **逻辑封装** (声明/表达式) → **参数操控** (arguments/Rest) → **匿名/自调用** (隔离与回调) → **现代方案** (箭头函数/默认参数)

### 📌 核心知识地图
1. **函数本质**：设计用来执行特定任务的可重用代码块。要求：简洁易读、可重复、易于测试。
2. **五种声明/创建形态**：
   - **函数声明**：`function f(){}`，支持变量提升（先调用后定义）。
   - **表达式函数**：将函数赋给变量 `const f = function(){}`，逻辑更严谨。
   - **匿名函数**：没有名字的“临时工”，常作为回调或表达式的值。
   - **自调用函数 (IIFE)**：`(function(){})()`，定义即执行，创建私有作用域，防止变量污染。
   - **箭头函数**：`() => {}`，现代 JS 标准，语法简洁，不绑定 `arguments`。
3. **参数处理的“双雄会”**：
   - **`arguments` 对象**：正则函数自带。**类数组对象**（非真数组），无法直接用数组方法。
   - **`...args` (剩余参数)**：箭头函数标配。**真正的数组**，可以使用 `sort`, `map` 等所有数组兵器。
4. **防御性编程**：
   - **默认参数**：`name = 'Peter'`。当调用者没给参数时，函数依然能稳健运行。

---

### 🔍 深度对比：拨开函数的迷雾

#### ⚡ `arguments` vs `...args`：谁才是真数组？

* **真相**：`typeof arguments` 是 `object`，但 `Array.isArray(arguments)` 是 `false`。
* **输入陷阱**：`sumAll([1,2])` 传入的是一个包装好的数组参数；`sumAll(1,2)` 传入的是两个独立的数字参数。

#### 🛡️ 表达式函数 vs 自调用函数 (IIFE)
* **函数表达式**：是**“抽油烟机”**，安好了不按开关（调用）不转。
* **自调用函数**：是**“烟花”**，点火（定义）即爆炸（执行），执行完即销毁。
* **本质区别**：表达式函数为了**复用**；自调用函数为了**隔离**（保护内部变量不被外界窥探）。

---

### 🧠 深度勾连与感悟
- **“内容与容器”的哲学**：
  - **感悟**：匿名函数是“内容”，表达式函数是“容器”。我们给“无名氏”安一个变量名，是为了在内存地址中给它留个位子，方便反复调用。
- **函数的“血统”鉴定**：
  - **勾连点**：使用 `Array.isArray()` 是识破类数组对象（如 `arguments`）伪装的唯一准则。
- **为什么箭头函数没有 `arguments`？**：
  - **感悟**：这是 JS 引擎的优化。箭头函数设计初衷就是轻量化。如果需要处理无限参数，强制要求使用更强大的 `...args` 真数组，这其实是倒逼开发者写出更规范的代码。
- **默认参数的稳定性**：
  - **感悟**：默认参数不仅是图省事，它更像是一种“合同保障”——确保函数内部在处理 `undefined` 时不会崩溃。

### 📌 核心知识地图

#### 1. 作用域 (Scope)：变量的“活动地盘”
* **全局作用域 (Global)**：声明在所有代码块外部，哪里都能访问。
* **局部作用域 (Local)**：被包裹在特定区域内。
* **⚠️ 关键字的“墙”**：
    * **`var` (函数作用域)**：只有 `function(){}` 能关住它，无视普通的 `{}`（如 if/for），容易造成变量污染。

#### 2. 对象 (Objects)：数据的“终极打包器”
* **创建**：`const obj = {}`。对象由“键 (Key)”和“值 (Value)”组成。
* **访问值的两套法则**：
    * **点语法 (`.`)**：`person.firstName`。适用于**规矩**的、无特殊字符的已知键名。

* **增改属性**：直接赋值即可，如 `person.isMarried = true` 或 `person.skills.push('Java')`。
* **四大法宝 (Object Methods)**：
    * `Object.assign({}, obj)`：**无损克隆**。把原对象的属性拷贝到一个全新的 `{}` 纸箱里，保护原数据。
    * `Object.keys(obj)`：抽取所有**键**，返回一个一维数组。
    * `Object.entries(obj)`：抽取**键和值**，返回一个二维数组 `[ [key, value], ... ]`。
    * `obj.hasOwnProperty('key')`：安全检查。判断对象肚子里到底有没有这个属性（返回 true/false）。

### 🧠 深度勾连与感悟

* **告别 `var` 的时代**：
    * **感悟**：理解了 `var` 的“穿墙术”，就能明白为什么现代前端开发要全量拥抱 `let` 和 `const`。把变量锁在它该呆的代码块里，是减少 Bug 的第一要义。
* **为什么 `'phone number'` 要加引号？**：
    * **避坑点**：JS 对象的键底层都是字符串。只要键名里出现了空格、连字符等“不合法”的变量命名符号，就必须穿上引号防弹衣，并且后续只能用 `['']` 来召唤它。
* **`Object.entries()` 的本质是“翻译”而非“复制”**：
    * **勾连点 (联动 Day 06)**：对象是铁板一块，没法直接用循环遍历。`entries()` 的伟大之处在于它把对象“拆解并翻译”成了二维数组，这样我们就能无缝接入 `for...of` 来批量处理对象里的数据了！

<summary><b>📅 Day 09：高阶函数 (Higher Order Functions) —— 逻辑的升维与自动化</b></summary>

### 📁 对应源码
- **路径**：[`9高阶函数.html`](9高阶函数.html)

> ### 💡 学习直觉：
> **函数即数据** (传参/返回) → **定时接管** (异步执行) → **数据流自动化** (Map/Filter/Reduce) → **声明式思维** (我只要结果，不管过程)

### 📌 核心知识地图

#### 1. 高阶函数 (Higher Order Function) 的定义
如果一个函数**接收另一个函数作为参数**（这种被当成参数的函数叫作**回调 Callback**），或者**返回一个函数**，那么它就是高阶函数。
* **特性**：它把函数当成了普通的变量在传来传去。

#### 2. 定时器：时间维度的高阶函数
* **`setTimeout(callback, ms)`**：**倒计时炸弹**。在指定的毫秒数之后，执行**一次**回调函数，然后结束。
* **`setInterval(callback, ms)`**：**无限节拍器**。每隔指定的毫秒数，就重复执行一次回调函数，直到被手动关停（`clearInterval`）。

#### 3. 函数式编程：数组的“自动化流水线”
这些是 JavaScript 数组自带的高阶方法。我们只需要提供“处理图纸（回调函数）”，剩下的遍历动作全部由内部自动完成。

| 方法 | 现实比喻 | 核心功能与返回值 | 备注特点 |
| :--- | :--- | :--- | :--- |
| **`forEach`** | **无情的打工人** | 遍历数组，对每个元素执行操作。**没有返回值** (undefined)。| 无法被 `break` 打断，就是替代普通 `for` 循环的简写。 |
| **`map`** | **加工厂** | 处理每个元素，并**返回一个长度相等的新数组**。 | 用于数据变形，比如全转大写、全部平方。 |
| **`filter`** | **保安 / 筛子** | 过滤出条件为 `true` 的元素，**返回一个新数组**。 | 原数组不变。条件写得很严谨时绝佳。 |
| **`reduce`** | **存钱罐 / 滚雪球** | 将所有元素揉成一个**单一值**（如求和、求乘积）。 | **极度重要**：必须提供初始值（如 `, 0`），否则遇空数组直接报错。 |
| **`every`** | **质检员 (一票否决)** | 检查是不是**所有**元素都满足条件，返回 `Boolean`。| 只要遇到 1 个不符合的，立刻刹车返回 `false`。 |
| **`some`** | **质检员 (一票赞成)** | 检查**有没有**元素满足条件，返回 `Boolean`。 | 只要遇到 1 个符合的，立刻刹车返回 `true`（短路机制）。 |
| **`find`** | **狙击手** | 寻找**第一个**满足条件的元素，**返回该元素本身**。 | 找到了就立刻停止，找不到返回 `undefined`。 |
| **`findIndex`**| **定位雷达** | 与 `find` 类似，但返回的是该元素的**索引号**。 | 找不到返回 `-1`。 |
| **`sort`** | **整理师** | 按升序或降序排列元素。 | ⚠️ **大坑**：默认把数字当成**字符串**来排序，且会修改原数组。 |

---

### 🧠 深度勾连与感悟

#### ⚡ 破译 `forEach` 的传参黑盒
* **感悟**：我们不需要手动给回调函数传参数（如 `element`）。因为高阶函数在底层接管了控制权，是它在内部循环时，**主动**把当前的元素、索引塞进了我们的回调函数里。我们只需“交出图纸”。

#### 🛡️ `reduce` 的保命符：初始值
* **避坑点**：`reduce((acc, cur) => acc + cur, 0)`，最后的这个 `, 0` 极其关键。如果不加初始值，一旦遇到空数组，JS 引擎不知道拿什么当“底薪”，会直接抛出致命错误。

#### ⚖️ `===` 与 `==` 的世纪决择
* **行业铁律**：在这些高阶函数（特别是 `filter`、`find`）写匹配条件时，**必须且只能使用 `===` (严格相等)**。`==` 会在后台偷偷转换数据类型（比如把 `0` 认作 `false`，把 `'1'` 认作 `1`），这会引发极为诡异的漏网之鱼或误伤。现代前端开发彻底抛弃 `==`。

<summary><b>📅 Day 10：集合与字典 (Set & Map) —— 唯一性与高级映射</b></summary>

### 📁 对应源码
- **路径**：[`10集合与地图.html`](10集合和映射.html)

> ### 💡 学习直觉：
> **去重思维** (Set) → **集合运算** (交/并/差) → **解构平铺** (Spread `...`) → **全能键对** (Map)

### 📌 核心知识地图

#### 1. 集合 (Set)：绝对唯一的“成员名单”
* **本质**：类似于数组，但**成员的值都是唯一的**，没有重复。
* **常用操作**：
    * `add(val)`：添加（重复则自动无视）。
    * `delete(val)`：删除。
    * `has(val)`：极速检查是否存在（性能远超数组 `includes`）。
    * `clear()`：清空全家桶。
    * **属性**：`.size`（注意不是 `.length`）。

#### 2. 扩展运算符 (`...`)：拆信封的魔术
* **作用**：将数组、集合等容器“拆开”，平铺成独立的元素。
* **用法场景**：
    * **合并**：`[...a, ...b]` 将两个容器的内容拆出来装进新数组。
    * **转换**：`[...mySet]` 快速将 Set 变回 Array。



#### 3. 集合间的逻辑运算
利用 `filter` 和 `has`，我们可以实现数学意义上的集合操作：
* **并集 (Union)**：`new Set([...a, ...b])`。
* **交集 (Intersection)**：`a.filter(x => bSet.has(x))`。
* **差集 (Difference)**：`a.filter(x => !bSet.has(x))`。

#### 4. 字典 (Map)：万物皆可为“键”
* **对比 Object**：普通对象的键只能是字符串；**Map 的键可以是任何类型**（数字、对象、函数等）。
* **常用操作**：
    * `set(key, val)`：存入。
    * `get(key)`：取值。
    * `has(key)`：判断键是否存在。
* **迭代**：支持 `for(const [key, value] of myMap)` 这种解构遍历。

---

### 🧠 深度勾连与感悟

#### ⚡ 唯一性的“判定准则”
* **感悟**：Set 认为 `5` 和 `5` 是重复的，但认为 `{}` 和 `{}` **不重复**。这是因为基本类型比对的是“值”，而引用类型（对象/数组）比对的是“内存地址”。两个空对象虽然长得一样，但灵魂（地址）不同。

#### 📦 为什么需要 `...` 扩展运算符？
* **感悟**：Set 虽然去重很爽，但它没有数组那种强大的 `map/filter` 武器。`...` 就像一座桥梁，让我们能把数据从 Set 里“倒出来”送进数组里进行高级加工，加工完再塞回 Set 去重。这种**“ Array ↔ Set ”**的往返跑是 JS 开发的常用套路。

#### 🗺️ Map：跳出字符串的限制
* **感悟**：Map 解决了 Object 的痛点。如果你需要给每一个 DOM 节点或者每一个函数绑定一份特定的配置数据，Map 是唯一的选择，因为它不会强行把你的键转成 `"[object Object]"`。

<summary><b>📅 Day 11：解构与扩展 (Destructuring & Spread) —— 数据的精准拆解与重组</b></summary>

### 📁 对应源码
- **路径**：[`11结构化与传播.html`](11结构化与传播.html)

> ### 💡 学习直觉：
> **数组解构** (靠座位号) → **对象解构** (靠暗号匹配) → **解构重命名** (防冲突避坑) → **Spread/Rest** (拆包与打包)

### 📌 核心知识地图

#### 1. 解构 (Destructuring)：精准提取数据
解构是一种从数组或对象中提取值，并将其赋给独立变量的优雅语法。

* **📦 数组解构（按顺序排队）**
  * **特点**：依靠索引顺序获取，变量名可以随便起。
  * **跳过元素**：用逗号 `,` 留空位。`let [num1, , num3] = [1, 2, 3]`。
  * **默认值**：防 `undefined`。`let [first = '默认名'] = []`。

* **🧩 对象解构（严丝合缝的暗号匹配）**
  * **🔥 铁律**：声明的变量名**必须**与对象里的键名（属性名）完全一致！
  * **🛡️ 避坑：变量名冲突与重命名**
    * *案发现场*：同时解构两个包含 `width` 属性的对象时，会触发 `Identifier 'width' has already been declared`。
    * *解法*：使用冒号 `:` 重命名。`let { width: w1 } = rect1;`（拿出来后改名叫 `w1`）。
  * **默认值**：`let { per2 = 60 } = rec2;`。



#### 2. 函数参数的直接解构 (高频实战操作)
不需要把整个对象传进函数再 `obj.width` 慢慢点，直接在参数括号里“拆箱”：
```javascript
const rect = { width: 20, height: 10 };
// 括号里直接解构，拿完就用！
const calcArea = ({ width, height }) => width * height;
```

#### 3. 扩散/扩展运算符 (Spread Operator `...`)
* **动作**：**“拆信封 / 平铺”**。把容器里的元素一个个掏出来。
* **核心场景**：
  1. **复制**：`const copy = [...arr]` 或 `const objCopy = {...obj}`（浅拷贝）。
  2. **合并**：`[...arr1, ...arr2]` 拼成一个大数组。
  3. **复制并修改**：`const newObj = {...user, age: 18}`（把 user 平铺，并强行覆盖或新增 age 属性）。

#### 4. 剩余参数 (Rest Parameter `...`)
当 `...` 出现在**解构的末尾**或**函数的参数括号里**时，它的动作变成了**“打包收尸”**。

* **🔥 绝对铁律：箭头函数中的强力吸尘器**
  * 在箭头函数中 `const sum = (...args) => {}`，`...args` 会把所有传进来的零散参数全部打包。
  * **返回值保证**：它在函数内部**百分之百、绝对是一个纯正的 Array（数组）**。彻底淘汰了老旧的 `arguments` 伪数组。你可以直接对它使用 `.map()` 或 `.reduce()`。



---

### 🧠 深度勾连与感悟

* **对象解构的本质**：
  * **感悟**：解构不是“引用”，而是“当场新建本地变量”。`const { width } = rect` 相当于在你的桌子上新建了一个叫 `width` 的变量。这就是为什么多次解构同名属性会报错冲突，因为桌子上不能有两个同名的常量。

* **`...` 的双面人生**：
  * 遇到 `...` 时，问自己一个问题：它现在是在**拆包**，还是在**打包**？在等号右边或者容器里就是 Spread（平铺拆包）；在等号左边或者函数形参里就是 Rest（打包收集）。

<summary><b>🟠 Day 12：正则表达式 (Regular Expressions) —— 文本的终极搜索法则</b></summary>

### 📁 对应源码
- **路径**：[`12正则表达式.html`](12正则表达式.html)

> ### 💡 学习直觉：
> **制造探测器** (定义模式) → **开启高级开关** (g/i/m) → **呼叫方法** (test/match) → **解密外星语** (元字符组合)

### 📌 核心知识地图

#### 1. 制造“超级探测器” (创建 RegExp)
正则表达式由**模式 (Pattern)** 和 **旗帜/标志 (Flags)** 组成。
* **写法 A（构造器）**：`new RegExp("love", "gi")` —— 适合模式是动态变量的情况。
* **写法 B（字面量）**：`/love/gi` —— 极其简洁，最常用。

#### 2. 探测器上的“高级开关” (Flags)
* `g` (Global)：**全局搜索**。不加它，找到第一个就停；加了它，全文地毯式搜索。
* `i` (Ignore case)：**大小写不敏感**。把 `A` 和 `a` 视作同一样东西。
* `m` (Multiline)：**多行模式**。让首尾锚点可以识别文本里的换行符。

#### 3. 核心干活方法 (Methods)
| 方法归属 | 代码语法 | 返回值 | 核心功能 |
| :--- | :--- | :--- | :--- |
| **RegExp 自带** | `pattern.test(str)` | `Boolean` | **"有没有？"** 检查文本是否符合规则。 |
| **String 自带** | `str.match(pattern)` | `Array / null` | **"给我挖出来！"** 返回所有符合规则的内容。 |
| **String 自带** | `str.search(pattern)`| `Index / -1` | **"在哪儿？"** 返回第一个匹配项的索引位置。 |
| **String 自带** | `str.replace(pattern, '新文本')` | `String` | **"全换掉！"** 批量替换敏感词或修改格式。 |

#### 4. 外星语密码本 (常用符号解密)



* **🎯 集合与基础设定**
    * `[]`：多选一集合。`[a-z]` 代表任意小写字母，`[Aa]pple` 匹配 Apple 或 apple。
    * `.`：匹配**除换行符以外**的任何单个字符。
    * `|`：或 (OR)。`apple|banana` 匹配苹果或香蕉。

* **🧙‍♂️ 魔法与转义 (`\`)**
    * `\d`：任意数字 (Digit)。等价于 `[0-9]`。
    * `\w`：字母、数字或下划线 (Word)。等价于 `[a-zA-Z0-9_]`。
    * `\.`：卸妆水，剥夺 `.` 的超能力，只匹配普通的句号。

* **⏱️ 量词 (控制出现次数)**
    * `*`：0 次或多次（可以没有，也可以无限多）。
    * `+`：1 次或多次（**至少得有 1 次**）。
    * `?`：0 次或 1 次（可选的）。
    * `{3}`：精准出现 3 次。`{3,8}` 出现 3 到 8 次。

* **⚓ 边界与定位**
    * `^`：**起始锚点**（绝对的开头）。
    * `$`：**结尾锚点**（绝对的结尾）。
    * `\b`：**单词边界**（隐形的门，不占字符长度）。



* **📦 魔法打包袋 `()`**
    * **组队**：`(ha)+` 让 "ha" 作为一个整体重复。
    * **捕获**：提取特定内容并在 `match` 数组中独立显示。

---

### 🧠 深度勾连与感悟

* **`^` 的“双面间谍”身份**：
    * 当它在方括号外面开头时：`/^This/`，它叫**起始锚点**，意思是“必须以 This 开头”。
    * 当它跑到方括号里面时：`/[^A-Za-z]/`，它立刻叛变变成了**“否定/取反”**，意思是“除了字母，其他都要”。

* **`\b` 零宽断言的顿悟**：
    * 算出 `/\b\w{4}\b/` 是 4 个字符而不是 6 个，是因为 `\b` 是一个不占物理座位的“边界检查门”。正则里的定位符只是用来确认环境，并不“吃掉”字符。

<summary><b>🟡 Day 13：控制台对象 (Console Methods) —— 代码世界的探照灯与透视镜</b></summary>

### 📁 对应源码
- **路径**：[`13控制台对象方法.html`](13控制台对象方法.html)

> ### 💡 学习直觉：
> **基础播报** (log/warn/error) → **格式化透视** (table) → **收纳整理** (group) → **性能与排雷** (time/assert)

### 📌 核心知识地图

#### 1. 基础播报台：四种颜色与语气
控制台不仅能打印，还能通过不同的级别在浏览器面板中进行过滤（Filter）。
* `console.log()`：**标准输出**。最常用的普通陈述句。
* `console.info()`：**提示信息**。在 Chrome 中和 log 几乎一样，在其他浏览器可能带有 ℹ️ 图标，用于播报程序状态。
* `console.warn()`：**黄色警报**。输出带黄色三角形的警告，不会阻断程序运行，但提醒你“这里可能有坑”。
* `console.error()`：**红色报错**。输出带红色叉号的错误信息，极其醒目。

#### 2. 占位符与彩蛋（字符串替换与 CSS 化装舞会）
除了直接打印变量，`console.log` 还支持类似 C 语言的占位符：
* `%d`（整数）、`%s`（字符串）：`console.log('%d %s', 30, 'Days')` 会拼接成 `30 Days`。
* **🔥 隐藏彩蛋 `%c`**：给文字穿上 CSS 衣服。遇到 `%c`，就会读取后面的 CSS 字符串。
  ```javascript
  // 打印一个绿底白字的 30% 标签
  console.log('%c30%', 'color: white; background: green;');
  ```

#### 3. 结构化透视：`console.table()`
数组和对象嵌套太深看不清？用 `table()` 直接生成带索引的二维表格。
* **适用场景**：包含多个对象的数组（比如从后端请求回来的用户列表）、二维数组、大型对象。

#### 4. 性能秒表：`console.time()` & `console.timeEnd()`
用来测试一段代码（比如复杂的 `for` 循环）到底跑了多少毫秒。
* **规则**：两者必须传入**完全相同的标签名**（比如 `'timer1'`）才能配对成功。

#### 5. 沉默的排雷兵：`console.assert()`
* **工作机制**：如果第一个参数的条件为 `true`，它什么也不干（保持沉默）；**只有当条件为 `false` 时，它才会跳出来抛出红色的报错！**
* **场景**：适合用来写测试用例，比如 `console.assert(user.age >= 18, '未成年人警告！')`。

#### 6. 抽屉式收纳：`console.group()` & `console.groupEnd()`
把杂乱的打印信息折叠成树状目录结构。
* `console.group('组名')`：拉开一个抽屉，接下来的打印都会自动缩进装进去。
* `console.groupEnd()`：关上抽屉。**千万别忘了写！**
* *(进阶技巧：使用 `console.groupCollapsed()` 可以让抽屉默认处于闭合状态，页面更清爽)*

---

### 🧠 深度勾连与感悟

* **不要过度依赖 `log`**：遇到复杂数据，第一反应应该是用 `table()`；遇到大量循环打印，第一反应应该是加 `groupCollapsed()`；遇到查性能瓶颈，直接上 `time()`。掌握这些工具，能让你在以后写 Vue 或 Node.js 项目时，查 Bug 的速度提升十倍。
* **前端的安全意识**：你在控制台能看到的东西，普通用户按 F12 也能看到。所以在项目最终上线（打包生产环境）之前，务必要清理掉所有敏感的 `console.log`，避免泄露内部逻辑。

<summary><b>🔵 Day 14：类与面向对象 (Classes & OOP) —— 构筑代码世界的造物法则</b></summary>

### 📁 对应源码
- **路径**：[`14类与面向对象.html`](14class.html)

> ### 💡 学习直觉：
> **Class** (造物图纸) → **Constructor** (出厂设置) → **Static** (问工厂要工具) → **Extends/Super** (继承老爸家产) → **Override** (子类青出于蓝)

### 📌 核心知识地图

#### 1. 类 (Class) 与 实例 (Instance)
* **概念**：`class` 是一张图纸，`new` 是根据图纸把东西真正造出来的动作。
* **构造函数 `constructor`**：每次 `new` 一辆新车（实例）时，第一步自动执行的函数。用来给这辆车分配专属的属性（比如颜色、车牌号 `this.color`）。

#### 2. 静态方法 (Static Methods) —— “挂在工厂大门上的工具”
* **定义**：在方法名前加 `static` 关键字。
* **调用方式**：**绝对不能**用 `new` 出来的实例调用，**只能用类名本身调用**。
* **核心场景**：不依赖特定对象数据的工具函数（比如获取当前时间、比较两个对象的大小）。
```javascript
class Person {
    static sayTime() {
        console.log(new Date());
    }
}
// ❌ 错误：const p = new Person(); p.sayTime();
// ✅ 正确：不需要 new，直接叫类名
Person.sayTime(); 
```

#### 3. 继承 (Inheritance) —— `extends`
让一个新类（子类）直接拥有另一个类（父类）的所有属性和方法，减少重复敲代码。
* **语法**：`class Child extends Parent {}`

#### 4. 呼叫老爸 (Super)
如果你在子类里写了 `constructor`，**必须**在第一行调用 `super()`！
* **翻译**：“在给自己置办新家当（`this.gender`）之前，必须先把我爸的家当（`first`, `last`）继承过来。”
```javascript
class Child extends Person {
    constructor(first, last, gender) {
        super(first, last); // 呼叫父类的 constructor，这句必须放在最前面！
        this.gender = gender; // 然后再添加子类特有的属性
    }
}
```

#### 5. 方法覆盖 / 重写 (Overriding)
子类觉得父类的方法不好用，可以直接写一个同名的方法把它**覆盖/替换**掉。
```javascript
class Person {
    introduce() { return '我是长辈'; }
}
class Child extends Person {
    introduce() { return '我是晚辈，时代变了'; } // 同名方法，直接覆盖！
}
```

---

### 🧠 深度勾连与感悟

* **`this` 到底是谁？** 记住铁律：谁调用了这个方法，`this` 就是谁。在普通实例方法里，`this` 就是那个被 `new` 出来的具体对象；但在 `static` 方法里，`this` 指向的是类本身（图纸）。
* **面向对象 (OOP) 的灵魂**：封装（把属性和方法包在 class 里）、继承（extends）、多态（方法覆盖）。这三大特性，是你未来进阶高级前端的基石。

## 🧰 JavaScript 核心避坑工具箱 (Day 09 - Day 11 升级版)

### 🚨 1. 遍历与高阶函数篇
* **❌ 陷阱：试图打断 `forEach`**
    * **案发现场**：在 `forEach` 里写了 `break` 或 `return`，以为能停下循环，结果它固执地跑完了全程。
    * **✅ 拆弹**：`forEach` 绝对无法中途刹车。需要提前刹车时，老老实实用回 `for...of`，或者使用自带短路机制的 `some()`（找 true 刹车）和 `every()`（找 false 刹车）。
* **❌ 陷阱：`reduce` 遇空报错**
    * **案发现场**：`arr.reduce((acc, cur) => acc + cur)`，当 `arr` 为空数组时，程序直接崩溃。
    * **✅ 拆弹**：永远记得给 `reduce` 加上初始值保命符。`arr.reduce(..., 0)`。

### 🚨 2. 逻辑与相等判断篇
* **❌ 陷阱：`==` 的“渣男”隐式转换**
    * **案发现场**：用 `filter(item => item.id == 0)`，结果把 `id` 为 `false` 或 `""` 的数据全抓出来了。
    * **✅ 拆弹**：彻底封杀 `==`。在现代前端开发中，**唯一真理是严格相等 `===`**，它绝不偷偷转换数据类型。

### 🚨 3. 解构与重命名篇 (Day 11 经典)
* **❌ 陷阱：对象解构变量名冲突**
    * **案发现场**：`const { width } = rect1; const { width } = rect2;` 触发 `SyntaxError: Identifier 'width' has already been declared`。
    * **✅ 拆弹**：解构等同于在本地当场声明新变量，不能重名。必须使用冒号重命名：`const { width: width2 } = rect2;`。
* **❌ 陷阱：以为对象解构能随便起名**
    * **案发现场**：`const { myName } = { firstName: 'Asab' }`，得到 `undefined`。
    * **✅ 拆弹**：对象解构是**“暗号匹配”**，变量名必须和对象里的键名一字不差！

### 🚨 4. 数据结构与内存篇 (Day 10 经典)
* **❌ 陷阱：Set 无法给对象去重**
    * **案发现场**：`new Set([{}, {}])`，你以为结果是 1 个空对象，其实 Set 里面存了 2 个空对象。
    * **✅ 拆弹**：Set 的唯一性对“引用类型（对象/数组）”看的是**内存地址**。两个长得一样的空对象，灵魂（地址）不同，Set 就会放行。
* **❌ 陷阱：老旧的 `arguments` 伪数组**
    * **案发现场**：在老式 `function` 里对 `arguments` 使用 `.map()`，直接报错找不到方法。
    * **✅ 拆弹**：全面拥抱箭头函数和**剩余参数 `...args`**。箭头函数的 `...args` 吸进来的数据，**百分之百是一个自带所有高级方法的真数组 Array**。

### 🚨 5. 异步定时器篇
* **❌ 陷阱：失控的节拍器 (内存泄漏)**
    * **案发现场**：开启了 `setInterval` 却不管它，导致后台无限循环，页面越来越卡。
    * **✅ 拆弹**：只要写了 `setInterval`，就必须立刻思考它的停止条件。用变量接住 ID (`const timer = setInterval(...)`)，并在适当时机 `clearInterval(timer)`。

### 🚨 7. 控制台调试篇 (Day 13 经典)
* **❌ 陷阱：`timeEnd` 找不到对应的计时器**
    * **案发现场**：`console.time('请求时间'); ...代码...; console.timeEnd('请求耗时');` 直接报错 `Timer '请求耗时' does not exist`。
    * **✅ 拆弹**：启动和停止秒表时，传入的字符串标签**必须一模一样**，连空格都不能差。
* **❌ 陷阱：抽屉合不上导致全盘缩进**
    * **案发现场**：写了 `console.group('用户列表')`，忘记写 `groupEnd`，导致后面所有的 log 全部缩进到了这个组里。
    * **✅ 拆弹**：像写括号 `{}` 一样，写下 `group()` 的那一刻，就顺手把 `groupEnd()` 写在下面。
* **❌ 陷阱：误会 `assert` 的触发时机**
    * **案发现场**：`console.assert(1 === 1, '验证成功')`，发现控制台什么都没打印。
    * **✅ 拆弹**：`assert` (断言) 是**“找茬大师”**。条件为真时它绝对闭嘴，只有条件为假（出现异常）时它才会大喊大叫。
